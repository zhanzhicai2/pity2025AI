# Pity项目新手接手指南

## 📚 项目简介

**Pity** 是一个基于 Python FastAPI + React 的 API 自动化测试平台，专注于接口测试的自动化管理。本项目提供了完整的测试用例管理、测试计划执行、定时任务调度等功能。

### 核心技术栈

**后端技术栈：**
- **Web框架**: FastAPI 0.75.0
- **数据库**: MySQL + SQLAlchemy 1.4.47 (ORM)
- **异步驱动**: aiomysql (异步MySQL操作)
- **缓存**: Redis 3.5.3
- **任务调度**: APScheduler 3.8.0
- **日志**: Loguru 0.6.0
- **Web服务器**: Uvicorn + Gunicorn

**前端技术栈：**
- **框架**: React 18 (基于 UmiJS 构建)
- **UI组件**: Ant Design Pro
- **构建工具**: Max build system
- **语言**: TypeScript/JavaScript

## 🔧 核心概念解析

### 1. MVC架构模式

本项目采用经典的MVC（Model-View-Controller）架构的变体：

- **Model（数据层）**: `app/models/` 目录下的数据库模型
- **View（视图层）**: 前端React页面
- **Controller（控制层）**: `app/routers/` 目录下的路由处理器
- **DAO（数据访问层）**: `app/crud/` 目录下的数据访问对象

**请求处理流程：**

```
用户请求 → 前端页面 → API路由 → 数据验证 → 业务逻辑 → 数据访问层 → 数据库
                      ↓
                 响应数据
```

### 2. 环境配置

项目支持多环境配置，通过环境变量 `pity_env` 控制：

```python
# config.py:116
PITY_ENV = os.environ.get("pity_env", "dev")
Config = ProConfig() if PITY_ENV and PITY_ENV.lower() == "pro" else DevConfig()
```

- **开发环境**: `pity_env=dev` 或不设置，使用 `conf/dev.env`
- **生产环境**: `pity_env=pro`，使用 `conf/pro.env`

### 3. 异步编程

项目大量使用Python的 `async/await` 异步编程模式：

```python
# 异步数据库操作示例（app/crud/UserDao.py）
async def get_user(session: AsyncSession, user_id: int):
    result = await session.execute(select(User).where(User.id == user_id))
    return result.scalars().first()
```

**为什么使用异步？**
- 提高并发处理能力
- 减少I/O阻塞等待时间
- 提升API响应速度

### 4. JWT认证

项目使用JWT（JSON Web Token）进行用户认证：

```python
# 登录成功后生成JWT token
token = create_access_token(data={"sub": user.id})

# 后续请求在Header中携带token
Authorization: Bearer <token>
```

JWT中间件会自动验证token并获取用户信息。

### 5. 权限系统

项目实现了三级权限控制：

```python
# config.py:47-50
MEMBER = 0  # 普通成员
MANAGER = 1  # 组长/管理者
ADMIN = 2    # 管理员
```

不同权限对应不同的操作权限。

### 6. 定时任务

使用APScheduler实现定时任务调度：

```python
# main.py:122-137
@pity.on_event('startup')
def init_scheduler():
    job_store = {
        'default': SQLAlchemyJobStore(url=Config.SQLALCHEMY_DATABASE_URI)
    }
    scheduler = AsyncIOScheduler()
    Scheduler.init(scheduler)
    Scheduler.configure(jobstores=job_store)
    Scheduler.start()
```

定时任务配置存储在数据库中，支持动态添加/修改/删除任务。

```python
from sqlalchemy import Column, Integer, DateTime
from app.models import Base


class UserStatistics(Base):
```

```python
    async def get_statistics(session: AsyncSession, user_id: int):
        result = await session.execute(
            select(UserStatistics).where(UserStatistics.user_id == user_id)
        )
        return result.scalars().first()

    @staticmethod
    async def update_login_count(session: AsyncSession, user_id: int):
        # 更新登录次数的逻辑
        pass
```

**步骤3：创建Schema（数据验证模型）**

```python
# app/schema/user_statistics.py
from pydantic import BaseModel
from datetime import datetime


class UserStatisticsSchema(BaseModel):
    id: int
    user_id: int
    login_count: int
    last_login: datetime

    class Config:
        orm_mode = True  # 支持ORM模式
```

**步骤4：创建路由处理器**

```python
# app/routers/statistics.py
from fastapi import APIRouter, Depends
from sqlalchemy.ext.asyncio import AsyncSession
from app.crud import get_session
from app.crud.UserStatisticsDao import UserStatisticsDao
from app.schema.user_statistics import UserStatisticsSchema

router = APIRouter()


@router.get("/statistics/{user_id}", response_model=UserStatisticsSchema)
async def get_user_statistics(user_id: int, session: AsyncSession = Depends(get_session)):
    """
    获取用户统计信息
    """
    stats = await UserStatisticsDao.get_statistics(session, user_id)
    return stats
```

**步骤5：注册路由**

```python
# 在 main.py 中添加
from app.routers.statistics import router as statistics_router

pity.include_router(statistics_router, dependencies=[Depends(request_info)])
```

## 🔍 常见问题FAQ

### Q1: 启动时报错 "Redis connect failed"

**A:** 检查Redis是否启动，或者关闭Redis（会导致定时任务可能重复执行）：

```bash
# 启动Redis
redis-server

# 或者在 dev.env 中设置 REDIS_ON=False
```

### Q2: 数据库表创建失败

**A:** 确保MySQL服务已启动，且数据库权限配置正确：

```sql
GRANT ALL PRIVILEGES ON pity.* TO 'root'@'localhost';
FLUSH PRIVILEGES;
```

### Q3: 前端无法连接后端API

**A:** 检查以下几点：
1. 后端服务是否启动（访问 `http://localhost:7777`）
2. 前端配置的baseURL是否正确（默认为 `http://localhost:8000`）
3. CORS配置是否允许前端域名（已配置为允许所有域名）

### Q4: 如何查看API文档？

**A:** FastAPI自动生成交互式API文档：

- Swagger UI: `http://localhost:7777/docs`
- ReDoc: `http://localhost:7777/redoc`

### Q5: 如何调试代码？

**A:** 推荐使用VSCode + Python插件进行调试：

1. 在VSCode中打开项目
2. 在代码行号左侧点击设置断点
3. 按 `F5` 或点击调试按钮
4. 选择 "Python File" 或创建 `.vscode/launch.json` 配置

**launch.json 配置示例：**

```json
{
    "version": "0.2.0",
    "configurations": [
        {
            "name": "Python: FastAPI",
            "type": "python",
            "request": "launch",
            "module": "uvicorn",
            "args": [
                "main:pity",
                "--host",
                "0.0.0.0",
                "--port",
                "7777",
                "--reload"
            ],
            "env": {
                "pity_env": "dev"
            }
        }
    ]
}
```
